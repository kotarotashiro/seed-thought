# 解読器 金標準サンプル（few-shot教材 兼 品質基準）

作成: 2026-07-08。Fable5クラスで生成した「解読器＋抽出器」の理想出力。
実装時はこのサンプルを few-shot として弱いモデル（Gemini Flash等）のプロンプトに埋め込む。

## 入力（保存された投稿）

```
最高の飲食店向けバナーができたかも！

この方(@azed_ai さん)を参考に海外で話題の「料理がバラバラに浮かぶ」プロンプトを、
ChatGPT Images 2でチラシやサイトのメインビジュアルにそのまま使える形に改良しました！

料理バナーのプロンプト+使い方です。

【プロンプト】
[subject], premium food advertising, white seamless background, high key studio
lighting, floating stacked composition, levitating product layers,
[ingredient_bits] scattered around the product, clean minimal layout, crisp soft
```

## 理想出力

### 一言でいうと

実写広告の定番「浮遊デコンストラクト」構図を、飲食店バナー用の穴埋め式プロンプトに実務変換した型。

### なぜすごいのか

- この「具材が層状に浮かぶ」構図はAI発祥ではなく、実写広告写真で確立された高級ジャンル
  （levitation / deconstructed food photography）の移植。見た人が「広告として見慣れた
  説得力」を感じる。本来はスタジオ撮影＋フードスタイリスト＋合成レタッチの外注領域。
- ただの作例紹介ではなく「チラシやサイトにそのまま使える形に改良」＝鑑賞用→実務用への変換。
  `white seamless background` と `clean minimal layout` は絵作りではなく後工程
  （文字入れ・切り抜き）のための指定で、ここが実務目線。
- `[subject]` と `[ingredient_bits]` の2箇所穴埋め式で、非デザイナーが自店メニューに
  置き換えるだけで使える再利用性。

### before → after

- **今まで**: この品質の浮遊系フードビジュアルは撮影・スタイリング・合成の外注領域で、
  小規模飲食店の選択肢はプロ発注か、ストックフォトの汎用画像で妥協するかだった。
- **これが出た**: 話題の構図が穴埋めプロンプトになり、ChatGPT Images 2で誰でも出せる形になった。
- **こう変わる**: 個人経営の飲食店が、自店の看板メニューで広告グレードのメインビジュアルを
  内製できる。「AI画像＝遊び」から「AI画像＝販促制作の実務」へ移る具体例のひとつ。

### 仕組みの解読（旧・学習カードの統合先）

| プロンプトの句 | 実務上の役割 |
|---|---|
| `premium food advertising` | ジャンル宣言。モデルに「広告写真」の絵作り全体（構図・質感・トーン）を引かせるアンカー |
| `white seamless background` | 白ホリ背景。切り抜き・文字載せ・チラシ流用のための指定 |
| `high key studio lighting` | ハイキー照明。影を薄くして清潔感と商業写真らしさを出す |
| `floating stacked composition` / `levitating product layers` | 本体のトリック。具材を層状に浮かせる非日常構図で視線を止める |
| `[ingredient_bits] scattered around` | 小物散らし。動きとシズル感を足しつつ「何の料理か」を伝える情報でもある |
| `clean minimal layout` | 余白の予約。キャッチコピーやロゴを入れるスペースを空けておく |

**原典・系譜**: 原典は @azed_ai の海外作例。構図の源流は実写広告の確立ジャンルで、
それをAIで再現→さらに実務用に改良、という二段の系譜。

### 型として抽出（資産庫に登録）

- **型名**: 浮遊デコンストラクト広告型
- **構造**: `[被写体]` ＋ ジャンル宣言 ＋ 背景（後工程用）＋ ライティング ＋ 構図トリック ＋ 小物散らし ＋ レイアウト配慮
- **可変スロット**: subject（看板メニュー等）/ ingredient_bits（構成素材）
- **転用範囲**: 「構成要素に分解できるプロダクト」全般。ジャンル宣言を
  `premium cosmetic advertising` に差し替えれば化粧品、飲料・雑貨にも。料理専用の型ではない。
- **使い方メモ**: 文字はプロンプトで入れず後工程で載せる前提の設計（そのための白背景と余白）

### 掛け合わせフック

`プロンプト設計` `商用ビジュアル` `飲食店販促` `構図の型` `要素列挙型プロンプト`

### 発信ネタの種（2パス生成への入力）

切り口: 「飲食店のバナー、今まではプロに頼むか素材サイトで妥協するかの二択だった」
→ before/after 構造がそのまま導入になる。処方箋型（読者がそのまま使える穴埋め
プロンプト付き）と相性が良い。

## 実行形（actionable）のジャンル適応原則 — 2026-07-08追加

応用アイデア（applicationIdeas.actionable）と隣接する型（adjacentPatterns.actionable）には
「そのまま使える実行形」を付けるが、**形式は投稿のジャンルに合わせる**。
プロンプトという形式を全ジャンルに機械的に当てはめない（気の利いた設計の核）。

| 投稿のジャンル | 実行形の形式 |
|---|---|
| プロンプト・テンプレ系 | そのまま貼れる具体プロンプト（元の構造を保ち差し替え箇所のみ変更） |
| 文章術・構文系 | 穴埋めテンプレ・書き換え例（[ ]で差し替え箇所を明示） |
| 思考法・フレームワーク系 | 自分の状況に当てる問いリスト（3〜5問）やチェックリスト |
| 手順・ワークフロー系 | 今日できる粒度の最小手順レシピ（3〜5ステップ） |
| ツール紹介・速報系 | 最初の15分でできる試し方 |
| 思想・エッセイ・感想 | null（無理に作らない） |

adjacentPatterns は解読の中で唯一、投稿の外の一般知識を使ってよいフィールド
（実在する定番のみ・でっちあげ禁止・検品パスで実在性チェック）。

## 弱いモデルでこの品質に近づける設計レバー

1. **1呼び出し1器官** — 解読/仕組み解剖/型抽出を1プロンプトに詰めない。2〜3段に分けて呼ぶ
   （Vercel 300秒制約内の直列で収める）。
2. **引用ファースト** — 最初に「投稿から根拠になる句を抜き出せ」を強制し、以降の全主張を
   抜き出した句に紐付けさせる。「なぜすごいのか」の空疎化を防ぐ最大のレバー。
3. **few-shot** — 本ファイルの理想出力をそのままプロンプトに埋め込む。
4. **フィールド単位のマイクロ指示＋文字数上限** — 例: 仕組み解剖は「各句について
   『実務上なんのための指定か』に答える。画風の形容で終えたら失格」。失敗モードを名指しで禁止。
5. **禁止則** — 数字の創作禁止（既存ルール踏襲）/ 根拠句のない「すごい」禁止 / 一般論の水増し禁止。
6. **2パス目＝検品** — チェックリストによる自己検証を1回
   （before/afterは投稿内の事実に基づくか / 解剖の各行は実務目的を言えているか）。
