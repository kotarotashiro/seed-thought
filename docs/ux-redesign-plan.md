# SeedThought UX 根本見直しプラン（v1 ドラフト）

作成: 2026-05-31 / 対象: 全画面の情報設計と学習体験の再設計
ゴール: 「やりたいことは今のまま、作り直すぐらいの気持ちで」を、リスクを抑えて段階的に実現する。

---

## 0. 前提・原則（ここを判断基準にする）

- **主目的 = 自分の理解を深める。** 副目的 = 理解したものを他者にも分ける。
  → 画面の主役は常に「理解」。共有・出力（SNS/コレクション/Notion）は主動線を邪魔しない位置へ。
- **投稿ごとの入力負担はかけない。** パーソナライズは「設定の思想/理想/属性」と保存履歴から効かせ、毎回の手入力を増やさない。
- **一人運用。** 設定の細かさより、迷わない一本道を優先する。

---

## 1. 現状の3つのズレ（診断）

1. **ナビが「機能」割りで「行為」割りになっていない。**
   現状トップ階層（Sidebar/MobileNav）: ホーム / 保存した投稿 / 学びメモ / コレクション / メモを検索 / 投稿に質問 / 保存傾向 / X下書き / 投稿を追加 + 設定3つ。
   → 9個がフラットに並ぶ。ユーザーの実際の流れは「**入れる → 理解する → 残す/見返す → 出す**」の一本なのに、ナビがその流れを表していない。

2. **「学ぶ」体験が1枚の長大カードに全部盛りで、密度が一定。**
   再設計で層は増えた（summary / originalIntent / capture / steps / beginnerZone / diagram / strict …）が、「**まず何を読めば“理解した”ことになるか**」の視線誘導がない。読む順序が設計されていない。

3. **副目的（共有・出力）が主目的の動線に混入している。**
   SNS出力・コレクション化・Notion同期は価値だが、「理解する」画面で主役級の面積を取り、集中を削ぐ。

---

## 2. 新しい情報設計（IA）: 4フェーズ

トップナビを **行為ベースの4つ** に再編する。各フェーズに既存ページを割り当て直すだけで、ページ自体の作り直しは最小限。

| フェーズ | 意味 | 主ページ | 含める既存機能 |
|---|---|---|---|
| **① 入れる (Capture)** | X取り込み・手動追加 | `/posts/new`, X連携, ブックマークレット | 投稿を追加 / X連携 / ブックマークレット |
| **② 理解する (Learn)** | 未学習を消化する場 | `/posts`（未学習を先頭に）→ `/posts/[id]/confirm` → `/posts/[id]/learning` | 保存した投稿 / 学習カード生成 / 本質を絞る / 投稿に質問 |
| **③ 残す・見返す (Keep)** | 蓄積した理解の再訪 | `/knowhow`, `/collections`, `/search` | 学びメモ / コレクション / メモを検索 |
| **④ 出す (Share)** | 他者に分ける（副目的） | 各カードの出力UI, `/drafts`, Notion同期 | SNS出力 / X下書き / Notion |

補助: **ホーム** は「今やるべき1件」を出すダッシュボード（未学習の件数・今日の1枚）。**保存傾向(`/insights`)** は ③ の中の分析タブに格納。**設定** はフッター/別枠。

### ナビ実装方針
- Sidebar/MobileNav を「9フラット」→「4フェーズ + ホーム + 設定」へ。
- 各フェーズはセクション見出し＋配下リンク（アコーディオン or グルーピング）。
- モバイルのボトムタブは ①〜④ + ホーム の5枠に収まる（現状の5枠とちょうど一致）。

変更ファイル: `src/components/layout/Sidebar.tsx`, `src/components/layout/MobileNav.tsx`（リンク構造の再編のみ。ページ実体は据え置きから着手できる）。

## 3. 学習画面の三段レイヤー化（②の中核）

`/posts/[postId]/learning` を「読む順序」が設計された3層に再構成する。データ構造は現状の `LearningOutput` のまま、**表示の優先度と開閉だけ**を変える（生成ロジック変更なし＝低リスク）。

```
┌─────────────────────────────────────────┐
│ 第1層：30秒で掴む（常時オープン・主役）        │
│  ・タイトル                                 │
│  ・一言でいうと（summary）                    │
│  ・投稿者の狙い（originalIntent）             │
│  ・なぜ見る価値があるか（whyForYou）           │
├─────────────────────────────────────────┤
│ 第2層：じっくり理解する（オープン）             │
│  ・投稿の中身そのもの（capture）＝忠実転写      │
│  ・初心者ゾーン（beginnerZone：詰まり＋用語）   │
├─────────────────────────────────────────┤
│ 第3層：深める・手を動かす（既定で畳む）          │
│  ・本質を絞る（strict learning, オンデマンド）  │
│  ・図解構造（diagram）                        │
│  ・自分用メモ（userMemo）                     │
└─────────────────────────────────────────┘
   ↓（主動線とは分離した帯）
   [ 出す ] SNS出力 / コレクションに追加 — 第3層の下、または右サイドに退避
```

設計ポイント:
- **第1層だけ読めば「理解した」状態になる**ことを担保する。ここに情報を集約。
- 第2・第3層は「もっと知りたい人向け」。第3層は折りたたみ既定（現状 `strictOpen` / `diagramOpen` の初期 false を踏襲）。
- 出力系（SNS/コレクション）は理解レイヤーから視覚的に分離（区切り帯＋トーンを落とす）。

変更ファイル: `src/app/posts/[postId]/learning/page.tsx`（セクション並び替え・グルーピング。新規データ取得は不要）。

## 4. ②で決めた設定簡素化との接続

- 設定は「デフォルトモデル1つ + APIキー」だけに簡素化済み（工程別割り当てUIは撤去）。
- 「ここぞのモデル変更」は **投稿詳細(`confirm`)の生成モデル選択** に一本化（既に動作）。
- → 新IAの ② フェーズ内で「学ぶ前にモデルを選べる」導線が自然に成立する。設計の整合が取れている。

## 5. 段階的ロールアウト（リスク順）

| Phase | 内容 | リスク | 備考 |
|---|---|---|---|
| **0（完了）** | 学びメモ一覧の「中心洞察」空表示バグ修正 / 設定の工程別UI撤去 | 低 | 本セッションで対応済み |
| **1** | ナビを4フェーズIAに再編（ページ実体は据え置き） | 低 | Sidebar/MobileNav のみ。即体感が変わる |
| **2** | 学習画面の三段レイヤー化 | 中 | 表示再構成のみ。生成ロジック不変 |
| **3** | ホームを「今やる1件」ダッシュボード化 / insights を ③ のタブへ統合 | 中 | 新規ページ要素あり |
| **4** | 出力系（SNS/コレクション/Notion）の動線分離・統一 | 中 | ④ フェーズへ集約 |

各 Phase は独立してリリース可能。1→2 だけでも体感は大きく変わる。

## 6. 未決事項・論点（要相談）

- **ホームの中身**: 「未学習の最新1件を即学ぶ」導線を主役にするか、件数サマリ中心にするか。
- **コレクションの位置づけ**: ③（見返す）と ④（出す）のどちらが主か。共有用途が強いなら ④ 寄りに。
- **検索の格納先**: 独立トップに残すか、③ 内の機能にするか（一覧と統合する手もある）。
- **モバイル5タブの割り当て**: ①〜④ + ホーム で確定か、ホームを外して「設定」を入れるか。
